# AGENTS.md

This repository is a simple, polished static site for the International Plant Pangenome Network.

## Primary goals
- Keep the site clean, spacious, restrained, and professional.
- Favour clarity, typography, whitespace, and alignment over decorative effects.
- Make the site feel credible for an academic/research audience.
- Preserve compatibility with GitHub Pages.
- Use the full name "International Plant Pangenome Network" as the primary brand.
- Avoid relying on the acronym "IPPN", which is already widely associated with the International Plant Phenotyping Network.
- If a short label is needed internally, use "IPPGN" sparingly, but do not make it the dominant public-facing brand.

## Information architecture
The site currently has four pages:
- Home
- How we work
- People
- Contact us

## Design direction
Borrow from:
- the restrained simplicity of Chandar Lab
- the professional, modern academic feel of UK-MAC

This means:
- generous whitespace
- tight, well-considered typography
- a calm neutral palette
- one muted accent colour only
- subtle borders/shadows only if genuinely useful
- no loud gradients, animations, or marketing-style graphics

## Logos
Reserve space for:
1. a future network logo
2. a future Earlham Institute logo

For now:
- use elegant text placeholders or empty framed blocks
- do not fabricate logos
- do not download third-party logo files automatically

## Content workflow
The user may wish to edit text before passing the site back to Codex.
Treat files in `docs/copy/` as the source-of-truth content drafts.
If asked to refine the design, preserve meaning and tone of the text unless explicitly told to rewrite.

## Technical guidance
- Keep this as a lightweight static site.
- Plain HTML/CSS/JS is preferred unless the user explicitly requests a framework.
- Make deployment easy on GitHub Pages.
- Keep filenames and structure simple.
- Ensure the navigation and layout are responsive.
- Avoid introducing build complexity unless there is a clear benefit.

## Tone
- professional
- collaborative
- open
- international
- not corporate
- not startup-like
